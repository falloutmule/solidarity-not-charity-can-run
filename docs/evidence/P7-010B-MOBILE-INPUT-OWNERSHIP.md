# P7-010B — Mobile input ownership audit

Baseline: `main` / `chromeinput2` (`aef4489` card baseline as supplied). This is a read-only source map; it does not choose a pointer, coalescing, capture, fallback, or sensitivity candidate.

Risk notation: **L1/L2/L3** = local, cross-file, or user-visible/high-impact change surface; **R** = likely regression vector; **U** = evidence/behavior uncertainty.

## Cross-file extraction seams

* `game-06` is the central owner for event registration, pointer/touch ownership state, coalesced processing, control geometry application, editor persistence, and the shared `inp`/`joy` state. Extracting only handlers would leave state and `crApplyPendingInputActions()` coupled to the fixed-step loop.
* `game-07` owns menu action dispatch and option presentation, but delegates all geometry and editor transitions to `game-06`; splitting these without a contract risks stale UI/layout keys.
* `game-14` initializes `options` before `game-06` uses sensitivity/size constants and persists a separate settings schema. It is also the compatibility boundary for legacy option fields.
* `game-22` owns portrait layout math and consumes both `options` and override helpers from `game-06`; it also publishes diagnostics. Layout extraction must preserve minimap/safe-area clamps and the fixed MENU band.
* CSS/HTML are static interaction owners: `touch-action:none`, no-scroll behavior, and stable IDs are prerequisites for the accepted Pointer-only Chrome route. Runtime extraction that changes IDs or CSS ownership invalidates the existing registration test assumptions.

## Boundary conclusion

The accepted `chromeinput2` contract is already explicit: Pointer Events are authoritative when available; Touch is a capability-gated fallback; coalesced samples are consumed exactly once; element capture owns active drags; MOVE and LOOK share one fixed-step input accumulator path. The highest-risk seams are changes that split listener registration from state cleanup, or geometry/editor persistence from layout computation. This audit intentionally leaves candidate selection and sensitivity/pointer/coalescing/capture policy unchanged.
